# Toronto Collision Risk Prediction & Safe Routing System

> COMP-4475 Topics in Artificial Intelligence — Accident Prediction Project

An end-to-end machine learning system that predicts collision fatality risk across Toronto and uses A\* pathfinding to compute safer alternative routes. Built using the City of Toronto's **Killed or Seriously Injured (KSI)** dataset (2006–2023).

---

## Overview

This project combines **supervised classification** with **spatial risk analysis** and **graph-based pathfinding** to answer two questions:

1. **Given a set of conditions (time, location, weather, road type), how likely is a collision to be fatal?**
2. **Can we route drivers through lower-risk areas with minimal extra travel distance?**

### Key Results

| Metric | Value |
|---|---|
| Best model | Random Forest |
| Accuracy | ~86.9% |
| F1 Score | ~0.823 |
| ROC-AUC | ~0.681 |
| Avg risk reduction (safe routing) | ~50% |
| Avg extra distance | ~38% |

---

## Features

- **3-Model Comparison** — Logistic Regression, Random Forest, and Gradient Boosting trained on ~18,000 deduplicated accident records
- **Temporal Feature Engineering** — Hour, day of week, season, rush hour, weekend flags
- **Spatial Risk Grid** — ~1km cells with per-cell collision density scores covering all of Toronto
- **Combined Risk Scoring** — Weighted blend of ML model probability (60%) and historical location risk (40%)
- **A\* Safe Routing** — Finds both shortest and safest paths between Toronto landmarks, comparing risk vs distance trade-offs
- **14 Publication-Ready Plots** — EDA, model evaluation, risk analysis, and routing visualizations
- **Interactive GUI** — Tkinter application for exploring safe routes between 15 Toronto landmarks

## Dataset

The **KSI (Killed or Seriously Injured)** dataset is published by the City of Toronto and contains records of traffic collisions where a person was killed or seriously injured. Each record includes:

- **Temporal**: Date, time of day
- **Spatial**: Latitude, longitude, district
- **Environmental**: Road surface condition, visibility, light conditions
- **Behavioral**: Speeding, aggressive driving, red light running, alcohol involvement
- **Involved parties**: Pedestrian, cyclist, automobile, motorcycle, truck
- **Outcome**: Fatal vs non-fatal (target variable)

The dataset spans **2006–2023** with approximately 18,000 unique accidents.

---

## Methodology

### Data Pipeline
- Rows with missing coordinates or dates are removed
- Out-of-bounds coordinates (outside Toronto) are filtered
- Multiple rows per accident (one per person involved) are deduplicated to prevent data leakage
- Binary flags use `max()` aggregation so that if *any* person in the accident had a flag (e.g., alcohol), it is preserved

### Models
- **Logistic Regression** — Baseline with `class_weight="balanced"` and StandardScaler
- **Random Forest** — 200 trees, max depth 15, balanced class weights
- **Gradient Boosting** — 200 trees, max depth 5, learning rate 0.1

### Safe Routing (A\*)
- A spatial risk grid (~500m cells) is built from collision data
- Each cell's route risk combines model-predicted risk (50%), fatality rate (30%), and collision volume (20%)
- A\* search finds both shortest-distance and lowest-risk paths
- Lake Ontario is modeled as impassable using a piecewise-linear shoreline approximation
- Risk heuristic is admissible (lower bound guarantee), ensuring optimal paths
